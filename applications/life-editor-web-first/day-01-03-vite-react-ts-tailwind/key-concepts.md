## CLI(ターミナル)の基本

### 必須 3 コマンド

| コマンド    | 意味                    | 用途                                     |
| ----------- | ----------------------- | ---------------------------------------- |
| `pwd`       | print working directory | 今いる場所を表示                         |
| `cd <path>` | change directory        | 移動。`cd ..` で 1 個上、`cd ~` でホーム |
| `ls -la`    | list -long -all         | 中身を詳細表示(隠しファイル含む)         |

**鉄則**: コマンドが期待通り動かない時は **まず `pwd` で現在地確認**。最頻出の原因が「居場所違い」。

### よく使うフラグ

- `-l` (long): 詳細表示
- `-a` (all): 隠しファイルも含む
- `-r` (recursive): 再帰的(サブフォルダも)
- `-h` (human): 人が読みやすいサイズ表記(KB / MB)

### `cat` vs `less` vs `head`

| コマンド        | 用途                                             |
| --------------- | ------------------------------------------------ |
| `cat file`      | ファイル全体を出力(短いファイル向け)             |
| `less file`     | ページ送りで読む(長いファイル向け、`q` で抜ける) |
| `head -20 file` | 先頭 20 行だけ                                   |
| `tail -20 file` | 末尾 20 行だけ                                   |
| `tail -f file`  | 末尾を追従表示(ログ監視によく使う)               |

### 打鍵ミスを減らす

- `↑` キー: 直前のコマンド呼び出し
- `Tab` キー: 補完(コマンド名・ファイル名・パス)
- `Ctrl+R`: 過去コマンド検索(逆方向インクリメンタル検索)

## npm の基礎

### `npm install` 系

| コマンド               | 意味                                |
| ---------------------- | ----------------------------------- |
| `npm install`          | package.json を見て依存を全部入れる |
| `npm install <pkg>`    | 本番依存に追加                      |
| `npm install -D <pkg>` | 開発依存に追加(`-D` = `--save-dev`) |
| `npm uninstall <pkg>`  | 削除                                |

**本番依存 vs 開発依存**:

- 本番依存(`dependencies`): アプリ実行に必要(React 等)
- 開発依存(`devDependencies`): 開発時だけ必要(TypeScript / Tailwind / テストツール等)

### `npm create <pkg>@<version>` の仕組み

```bash
npm create vite@latest . -- --template react-ts
```

を分解すると:

| 部品                  | 意味                                           |
| --------------------- | ---------------------------------------------- |
| `npm create`          | プロジェクト雛形作成サブコマンド(= `npm init`) |
| `vite@latest`         | `create-vite` パッケージの最新版を取って実行   |
| `.`                   | 雛形展開先 = 今のディレクトリ                  |
| `--`                  | ここから右はツール本体への引数                 |
| `--template react-ts` | create-vite への "React + TS テンプレ" 指示    |

`npm create vite` のように `create-` プレフィックスを書かないのが慣例。

### npm エラーパターン

| エラー                | 典型原因                                                                 |
| --------------------- | ------------------------------------------------------------------------ |
| `404 Not Found`       | パッケージ名のタイポ(`@teilwindcss` → `@tailwindcss` 等)                 |
| `EACCES`              | 権限エラー。`sudo` ではなく **nvm でユーザ領域に Node を入れる**のが正解 |
| `ENOENT package.json` | カレントディレクトリに package.json が無い(居場所違い)                   |
| `peer dep` 警告       | 互換バージョンの組み合わせ問題。多くは無視可                             |

## Vite / React / TypeScript / Tailwind の役割

### ひとこと要約

| ツール     | 一言                                   |
| ---------- | -------------------------------------- |
| Vite       | "コードを束ねて配るやつ"               |
| React      | "画面を関数(コンポーネント)で書く流儀" |
| TypeScript | "型を足した JS"                        |
| Tailwind   | "className でスタイルを当てる流儀"     |

### Vite の 2 モード

- `npm run dev`: 開発サーバ(http://localhost:5173)。HMR でファイル保存時に即反映
- `npm run build`: 本番用 `dist/` を生成。これを Web サーバや Capacitor に置く

### TypeScript の重要な勘違いポイント

- TS を書いても **ブラウザで動くのは結局 JS**
- TS は「書いてる時に型チェック → ビルド時に型を消した JS にする」
- 実行性能は JS と同じ
- 「型を書く労力」と「実行前にバグを潰せる安心」のトレードオフ
